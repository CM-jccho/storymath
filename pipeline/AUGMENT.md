# 문제 증량 공통 지침 (단원당 3문제 → 6문제)

담당 단원 파일 각각에 대해:

1. 기존 파일을 **전부 읽고** 기존 3문제의 소재·수 범위·어휘 수준·단계 설계를 파악한다.
2. 문제 3개를 **추가**한다 — 난이도별 1개씩, id는 `<unitId>-d1-02`, `<unitId>-d2-02`, `<unitId>-d3-02`.
3. 새 문제 규칙:
   - 기존 문제와 **같은 단원 범위·같은 학년 수준** (기존 문제가 쓰는 연산·개념 범위를 벗어나지 말 것)
   - 기존 문제와 소재·스토리·수치 패턴 **중복 금지** (다른 하위 개념이나 다른 상황을 다룰 것)
   - 난이도 보정: 기존 d1/d2/d3와 같은 수준으로 (d1=한 단계 적용, d2=두 가지 조합, d3=전략·조건 해석)
   - Polya 4단계(이해→계획→실행→반성) steps, 기존 파일과 같은 스타일
   - 텍스트 전용, input 답은 정수·소수 문자열만(콤마 없이), 분수·식·근호 답은 single 선택형
   - 모든 계산 두 번 검산
4. **금지**: 기존 3문제 수정, standards 수정, 단원 메타데이터(unitId·unitName·domain 등) 수정.
5. problems 배열 안에서 난이도 순(d1-01, d1-02, d2-01, d2-02, d3-01, d3-02)으로 정렬해 저장.
6. 작성 후 필수: `python3 pipeline/validate.py <unitId ...>` — 오류 0까지 수정.

최종 응답: 단원별 추가한 문제 제목·난이도·핵심 계산·정답 목록과 검증 결과만 간단히.
