# 콘텐츠 생성 파이프라인 — 다른 학년으로 확장하기

초4는 이 파이프라인으로 제작 완료. 새 학년을 열 때 아래 절차를 따른다.

## 절차

### 1. 교육과정 조사
- 해당 학년의 2022 개정 교육과정 단원 구성을 확인한다 (교과서 목차 + NCIC 국가교육과정정보센터 성취기준).
- `data/curriculum.json`의 해당 학년에 `semesters` 배열을 채우고 `status`를 `"active"`로 바꾼다.
- 단원 id 규칙: `<학년id>-<학기>-<단원번호>` (예: `e5-1-2`, `m1-2-3`).

### 2. 문제 생성 (Claude에게 맡기기)
Claude Code에서 단원 묶음별로 에이전트를 병렬 실행한다. 프롬프트 골격:

```
당신은 한국 초등/중등 수학 문제 콘텐츠 저작자입니다.
먼저 pipeline/SCHEMA.md와 data/problems/e4-1-3.json(예시)을 읽고 형식을 익히세요.
그다음 아래 단원 파일을 data/problems/에 작성하세요:

**<unitId>.json — <학년> <학기> <단원명> (domain: <영역>)**
범위: <이 단원에서 다루는 개념을 구체적으로. 상위 학년 개념 금지 항목도 명시>

공통 규칙:
- 단원마다 문제 3개: difficulty 1(기본)/2(응용)/3(심화)
- Polya 4단계(이해→계획→실행→반성) steps 구조, 예시 파일의 단계 설계 방식을 따를 것
- 스토리 기반 서사형 문제, 해당 학년 수준 어휘
- 텍스트 전용 (그림 없이 완전히 풀 수 있어야 함; 자료는 story 안에 \n 목록으로)
- input 답은 정수/소수 문자열만, 복잡한 답은 선택형으로
- 모든 계산을 두 번 검산
작성 후: python3 pipeline/validate.py <unitId ...> 를 오류가 없어질 때까지 실행
```

핵심 요령:
- **단원 범위를 프롬프트에 명시**하는 것이 품질의 절반이다. "이 학년에서 아직 안 배운 것"을 금지 항목으로 적어라.
- 도형·그래프 단원은 텍스트로 기술 가능한 소재(카드 뒤집기, 측정값 목록 등)를 프롬프트에서 예시로 제안하라.
- 에이전트 하나당 2~3단원이 적당하다.

### 3. 기계 검증
```bash
python3 pipeline/validate.py          # 전체
python3 pipeline/validate.py e5-1-1   # 특정 단원
```
스키마·난이도 구성·단계 순서·인덱스 범위를 검사한다.

### 4. 내용 검수 (사람 + AI 교차)
기계 검증은 숫자가 맞는지 모른다. 반드시 별도로:
1. **AI 교차 검증**: 생성에 참여하지 않은 새 Claude 세션에 "이 파일의 모든 문제를 실제로 풀어서 answer·finalAnswer가 맞는지, 학년 범위를 벗어나지 않는지 검증하라"고 시킨다.
2. **성취기준 대조**: standards의 코드·문구를 NCIC 원문과 대조한다.
3. **샘플 플레이**: 사이트에서 직접 몇 문제를 풀어 본다. 특히 힌트가 답을 노출하지 않는지, 오답 보기가 그럴듯한지.

### 5. 배포
정적 파일뿐이므로 전체 폴더를 그대로 올리면 된다 (GitHub Pages, Vercel 등).
로컬 확인: `python3 -m http.server` 후 http://localhost:8000

## 성취기준 원문 확보 방법 (표준 절차)
KICE 학생평가지원포털 REST API가 가장 확실하다:
```
GET https://stas.moe.go.kr/rest/acvmt/acvmtStd/acvmtStdList?sCond=CD&sWord=<코드접두어>&size=300
```
- `sWord`: `2수`(초1~2), `4수`(초3~4), `6수`(초5~6), `9수`(중1~3), `10공수1`/`10공수2`(고1) 등
- 응답 `content[]`에서 **`eduCurclmNm == "2022년 개정"` 필터 필수** (2015 개정이 섞여 옴)
- 고교 코드는 3단 형식([10공수1-01-01])임에 주의
- 2026-07-02: 초3·초6 각 36문제 전수 교차검증 완료 (학년별 독립 AI 검증자 재풀이) — CRITICAL/MAJOR 0건. 초6 MINOR 1건(정답 노출 힌트) 수정 반영. 성취기준은 standards-e3/e6.json(공식 원문) 생성 단계 적용.
- 2026-07-02: 초5 36문제 전수 교차검증 완료 (독립 AI 검증자 재풀이) — CRITICAL/MAJOR 0건, MINOR 1건(문구) 수정 반영. 성취기준은 pipeline/standards-e5.json(공식 원문)을 생성 단계부터 사용.
- 2026-07-02: 초4 36문제 전수 교차검증 완료 (독립 AI 검증자가 전 문제 재풀이) — 계산·논리·학년범위·힌트 오류 0건
- 2026-07-02: 초4 standards 코드·문구를 2022 개정 공식 성취기준 원문으로 일괄 정정 (출처: 경남교육청 "2022 개정 교육과정에 따른 성취기준별 성취수준(초등)" 3~4학년군 문서, 교육부 고시 제2022-33호 기반)
- 3~4학년군 공식 성취기준 전체 목록은 위 문서 참조. 새 학년(5~6학년군 [6수XX-YY], 중학교 [9수XX-YY]) 확장 시 같은 방식으로 해당 학년군 원문을 확보해 기입할 것.
