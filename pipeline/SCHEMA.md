# StoryMath 문제 데이터 스키마

단원별로 `data/problems/<unitId>.json` 파일 하나. `unitId`는 `curriculum.json`의 단원 id와 일치해야 한다 (예: `e4-1-3` = 초4 1학기 3단원).

## 단원 파일 구조

```json
{
  "unitId": "e4-1-3",
  "gradeId": "e4",
  "semester": 1,
  "unitName": "곱셈과 나눗셈",
  "domain": "수와 연산",
  "standards": [
    { "code": "[4수01-10]", "text": "성취기준 요지 (검수 시 NCIC 원문 대조)" }
  ],
  "problems": [ /* 아래 문제 객체, 난이도 1→3 순서 */ ]
}
```

## 문제 객체

```json
{
  "id": "e4-1-3-d1-01",
  "difficulty": 1,
  "title": "짧은 제목",
  "story": "서사가 있는 문제 상황. 주인공 이름, 맥락, 숫자가 자연스럽게 들어간 2~4문장.",
  "question": "최종적으로 구해야 하는 것을 명시한 한 문장 질문.",
  "steps": [ /* 아래 단계 객체, 순서대로 진행 */ ],
  "finalAnswer": "12상자",
  "wrapUp": "이 문제에서 배운 풀이 전략을 1~2문장으로 정리."
}
```

- `id` 형식: `<unitId>-d<난이도>-<일련번호 2자리>`
- 난이도: 1 기본(한 단계 적용), 2 응용(두 가지 이상 조합), 3 심화(전략 수립·해석 필요)

## 단계(step) 객체 — Polya 4단계 프레임워크

모든 문제는 **이해 → 계획 → 실행 → 반성** 순서를 따른다. 응용·심화 문제는 계획→실행 사이클을 2번 이상 반복할 수 있다.

```json
{
  "phase": "이해",
  "type": "multi",
  "prompt": "학습자에게 보여줄 질문",
  "options": ["보기1", "보기2", "보기3", "보기4"],
  "answer": [0, 1],
  "hint": "막혔을 때 보여줄 힌트 (답을 직접 말하지 말고 생각의 방향만)",
  "explain": "정답 후 보여줄 설명 — 왜 이것이 맞는지, 다음 단계와 어떻게 이어지는지"
}
```

| 필드 | 값 | 설명 |
|---|---|---|
| `phase` | `이해` `계획` `실행` `반성` | Polya 단계. UI에 단계 배지로 표시 |
| `type` | `multi` `single` `input` | multi=복수 선택, single=단일 선택, input=직접 입력 |
| `answer` | multi: 인덱스 배열 / single: 인덱스 / input: 문자열 | input은 공백 제거 후 비교, 숫자면 수치 비교 |
| `unit` | 문자열 (input 전용, 선택) | 입력창 뒤에 붙는 단위 (예: "상자") |

## 단계 구성 원칙

1. **이해**: 최소 2단계 — "알고 있는 것"(multi) + "구해야 하는 것"(single). 오답 보기에는 문제에 없는 그럴듯한 정보를 섞는다.
2. **계획**: "그럼 먼저 뭘 해야 할까?" — 연산/전략 선택. 오답 보기는 아이들이 실제로 헷갈리는 연산(곱↔나눗셈 등)으로 구성하고, explain에서 왜 그 전략인지 반드시 설명한다.
3. **실행**: 계획에서 고른 연산을 실제 수행(input 권장). 심화는 중간값을 나눠 여러 실행 단계로 쪼갠다.
4. **반성**: 검산 방법 선택, 답의 의미 해석(나머지 처리, 단위 확인 등). "답이 말이 되는가?"를 묻는다.

## 품질 체크리스트 (검수용)

- [ ] 숫자 계산이 전부 맞는가 (단계 답, finalAnswer 일치)
- [ ] 해당 학년 교육과정 범위를 벗어난 개념·연산이 없는가
- [ ] 힌트가 정답을 직접 노출하지 않는가
- [ ] 오답 보기가 그럴듯한가 (무의미한 보기 금지)
- [ ] 스토리가 자연스럽고 숫자가 현실적인가
- [ ] 모든 step의 answer 인덱스가 options 범위 안인가
